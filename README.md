# Agentic AI and Data Architecture in Aviation

This repository contains several book projects in Markdown format and tools to convert them to HTML.


Agentic AI refers to artificial intelligence systems capable of autonomous decision-making and action, learning from their environment without human intervention. In aviation, this could include AI managing flight paths, predicting maintenance issues, or automating customer service. Enterprise data architecture, on the other hand, involves designing and managing organizational data systems to ensure accuracy, accessibility, and security, crucial for handling vast data from flight operations, passenger information, and maintenance records.

The aviation industry generates significant data, with commercial aircraft producing up to 20 terabytes per engine per flight hour, as noted in a 2022 article by Thoughtworks ([Data mesh Tackling five of the aviation industry’s biggest data challenges | Thoughtworks](https://www.thoughtworks.com/insights/articles/data-mesh-tackling-five-of-the-aviation-industry-s-biggest-data-)). This underscores the need for advanced data architectures to support AI applications, such as predictive maintenance and real-time decision-making, enhancing efficiency and safety.

### Detailed Volume Descriptions

1. **Foundations of Agentic AI and Data Architecture in Aviation**: This volume introduces agentic AI, emphasizing its autonomous capabilities, and explores enterprise data architecture, referencing standards like the IATA Industry Data Model ([IATA - Airline Industry Data Model](https://www.iata.org/en/about/corporate-structure/passenger-standards-conference/architecture-technology-strategy/industry-data-model/)). It discusses challenges such as data interoperability and regulatory compliance, setting the stage for subsequent volumes.
2. **Agentic AI in Aviation Operations**: Focuses on how AI agents optimize flight operations, including path optimization and air traffic management, as highlighted in a 2024 Forbes article ([Navigating The Skies With AI: How Airlines Are Transforming Air Travel](https://www.forbes.com/sites/neilsahota/2024/03/29/navigating-the-skies-with-ai-how-airlines-are-transforming-air-travel/)). It covers real-time decision-making, with examples from Airbus's AI programs ([Artificial intelligence | Airbus](https://www.airbus.com/en/innovation/digital-transformation/artificial-intelligence)).
3. **Maintenance and Health Management with AI**: Details predictive maintenance, where AI analyzes sensor data to predict failures, as seen in a 2024 Medium article ([Top 5 Gen AI Use Cases in Aviation | by Edwin Lisowski | Medium](https://medium.com/@elisowski/top-5-gen-ai-use-cases-in-aviation-20b62583dbfa)). It discusses autonomous scheduling and integration with IoT, enhancing aircraft utilization rates.
4. **Enhancing Customer Experience through AI**: Explores AI-driven personalization, such as Delta's use of AI for in-flight entertainment recommendations, and chatbots like Singapore Airlines' Kris, as noted in a 2024 AIOLA blog ([The Future of AI in Aviation](https://aiola.ai/blog/future-of-ai-in-aviation/)). It addresses data privacy, crucial for passenger trust.
5. **Building and Managing AI-Enabled Data Architectures**: Focuses on designing scalable data platforms, referencing Microsoft's Azure solutions for aviation ([AI solutions and data platforms for the aviation industry - Microsoft Industry Blogs](https://www.microsoft.com/en-us/industry/blog/manufacturing-and-mobility/2024/10/09/ai-solutions-and-data-platforms-for-the-aviation-industry/)). It covers tools like data mesh and security measures, ensuring future-proof systems.
6. **Safety and Security in AI-Integrated Aviation Systems**: Discusses AI's role in anomaly detection and emergency responses, with insights from EASA's AI roadmap ([Artificial Intelligence and Aviation | EASA](https://www.easa.europa.eu/en/light/topics/artificial-intelligence-and-aviation-0)). It addresses regulatory compliance and ethical considerations, vital for safety-critical operations.
7. **Optimizing Supply Chain and Logistics with AI**: Examines AI's role in managing inventory and logistics, as seen in a 2019 Extrica article on big data in aircraft health management ([Application architecture and typical cases of big data technology in health management of civil aircraft system - Extrica](https://www.extrica.com/article/20428)). It highlights operational resilience, often overlooked in customer-focused discussions.
8. **Human-AI Collaboration in Aviation**: Explores collaboration models, including training programs for pilots and interface design, as discussed in a 2025 AVweb article on AI in avionics ([AI in Avionics - AVweb](https://www.avweb.com/features/ai-in-avionics/)). It ensures effective human-AI interaction, crucial for adoption.
9. **Ethical Considerations in Aviation AI**: Addresses bias, accountability, and transparency, drawing from a 2024 Springer article on AI in aviation safety ([Artificial Intelligence in Aviation Safety: Systematic Review and Biometric Analysis | International Journal of Computational Intelligence Systems](https://link.springer.com/article/10.1007/s44196-024-00671-w)). It discusses strategies for ethical AI deployment, especially in passenger data analysis.
10. **Case Studies and Best Practices in Aviation AI**: Compiles real-world examples, such as Boeing's AI-powered robots and Delta's revenue management systems, as seen in a 2024 Symphony Solutions article ([AI Revolution in Aviation: Shaping the Future of Air Travel | Symphony Solutions](https://symphony-solutions.com/insights/ai-in-aviation)). It shares lessons learned, offering practical insights for implementation.

## Converting Books to HTML

### Using the Unified Converter Script

The recommended way to convert any book to HTML is using the `convert_book_to_html.py` script:

```bash
python convert_book_to_html.py <book_directory> [options]
```

#### Parameters:

- `book_directory`: Path to the book directory containing Markdown files
- `-o, --output-dir`: (Optional) Directory to save the HTML output file (defaults to "output" directory)
- `-v, --version`: (Optional) Version string to include in the HTML title (default: v1.0)

#### Examples:

Convert the agentic AI book with default settings:
```bash
python convert_book_to_html.py agentic_ai_modern_data_architecture
```

Convert with custom version number:
```bash
python convert_book_to_html.py agentic_ai_modern_data_architecture -v v2.1
```

Specify a custom output directory:
```bash
python convert_book_to_html.py agentic_ai_modern_data_architecture -o /path/to/output
```

### Features

- **Automatic Table of Contents** - Creates a clickable table of contents based on chapter files
- **Mermaid Diagram Support** - Properly renders Mermaid diagrams found in Markdown files
- **Responsive Design** - HTML output is styled to work well on different screen sizes
- **Code Highlighting** - Properly formats code blocks
- **Clean Navigation** - Chapters are clearly separated with navigation anchors

## HTML Output

The HTML output includes:
- Responsive styling for better readability
- Proper rendering of Mermaid diagrams
- Table of contents with links to each chapter
- Code syntax highlighting
- Clean, professional formatting

The output file will be saved to the specified output directory (or the default "output" directory) with a filename based on the book directory name and version.
